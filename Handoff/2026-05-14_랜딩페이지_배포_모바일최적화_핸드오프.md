## 3. 이번 세션 작업 내역

### 3-1. 모바일 레이아웃 전면 수정
CSS `@media` 추가 위치: HTML 파일 `</style>` 직전

| 문제 | 수정 내용 |
|---|---|
| 헤더 100px 너무 높음 | 모바일 `:root { --header-h: 64px }` |
| 히어로 배지 4개 가로 overflow | `flex-wrap: wrap` + `flex: 0 0 calc(50% - 4px)` → 2×2 그리드 |
| 화살표 아이콘 모바일 방해 | `.hero-badge-arrow { display: none }` |
| 워크플로우 제목 32px | 모바일 20px |
| 워크플로우 캡션 20px | 모바일 13px |
| values-row 3컬럼 | 모바일 `flex-direction: column` |
| agent-card min-height: 340px | 모바일 `min-height: auto` |
| 860px 태블릿 배지 | 폰트 12px, 아이콘 28px 축소 |
| 380px 초소형 | 헤딩 20px, 배지 폰트 11px |

### 3-2. 플레이 버튼 iOS 겹침 해결
**원인:** `<video controls>` 속성이 있으면 iOS Safari가 중앙에 네이티브 원형 플레이 버튼을 렌더링 → 커스텀 오렌지 버튼과 겹쳐 보임

**3중 수정:**
1. **HTML:** `<video>` 태그에서 `controls` 속성 제거 (초기 로드 시 네이티브 버튼 원천 차단)
2. **JS:** 커스텀 버튼 클릭 시 `heroVideo.setAttribute('controls', '')` 추가 → 재생 중 seek/pause/fullscreen 정상 사용 가능. 영상 종료 시 `heroVideo.removeAttribute('controls')` + `heroVideo.load()`로 초기화
3. **CSS:** `video::-webkit-media-controls-start-playback-button { display: none !important }` 백업 방어

### 3-3. 퍼포먼스 최적화

| 항목 | 조치 |
|---|---|
| `workflowBeamSpin` (conic-gradient 회전) | `will-change: transform` 추가 (GPU 레이어 분리), 모바일에서 `animation: none` |
| 폴드 하단 이미지 6개 (4MB+) | `loading="lazy" decoding="async"` |
| 폴드 위 이미지 (로고, 배지 4개) | `decoding="async"` (즉시 로드 유지) |
| 데모영상 | 기존 `preload="metadata"` 유지 (버튼 클릭 전 다운로드 없음) |
